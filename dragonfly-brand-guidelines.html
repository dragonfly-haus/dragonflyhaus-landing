01 — Identity

dragonfly is a software factory. We build special solutions for clients who find us. We don't advertise, we don't pitch. The brand reflects this: geometric, silent, precise.

The mark is a dragonfly constructed from four geometric shards, a body dot, and a tail line terminating in a point. Front wings render at full opacity; back wings at 25% — creating depth from a flat shape.

02 — Logo construction

Anatomy

Back wings: Smaller triangles, offset lower, opacity 25%

03 — Color

04 — Typography

Two typefaces. Monospace leads, serif supports.

Primary: JetBrains Mono
Fallbacks: SF Mono, Fira Code, Cascadia Code, monospace
Weights: 400 (regular), 500 (medium — headings only)
Letter-spacing: 2-6px for brand text, 0 for body copy
Use for: Wordmark, UI elements, navigation, terminal, labels, code
Secondary: Instrument Serif (italic)
Fallbacks: Georgia, Times New Roman, serif
Weight: 400 only
Use for: Editorial prose, manifesto copy, descriptive text
Purpose: Creates tension between the technical monospace voice and a human, literary register. The contrast is intentional — code meets manifesto.

06 — Clear space

Maintain a clear zone around the mark equal to the width of one front wing. No text, graphics, or edges should enter this zone.

07 — Digital mark treatment

In digital/interactive contexts, the mark comes alive. These treatments are permitted only on screen, never in print or static assets.

Wing coloring: Front wings in accent-bright (#6fbfaa dark / #4a9a84 light), back wings in accent (#5b8c7e dark / #3d7a6a light). Body and tail remain currentColor.
Wing shimmer: Slow hue-rotate + brightness animation (8s cycle). Subtle iridescence, not a color show.
Wing beat: Gentle scaleX breathing (4s cycle, 0.92 minimum). Adds life without distraction.
Float: Vertical oscillation (6s, 0.5rem amplitude). The mark hovers, never sits static.
Hover state: Wings brighten to 0.9 / 0.4 opacity on hover.
Radial glow: Faint teal gradient behind the mark. Atmosphere, not spotlight.

08 — Ambient elements

The landing page uses ambient visual systems that extend the brand into environmental design. These are part of the brand language for digital surfaces.

Particle field: Drifting teal dots with connecting lines between nearby particles. Evokes a living network — decentralised, in motion, interconnected. Particles react to cursor proximity with a gentle push.
Mouse glow: Soft radial gradient following the cursor. Creates a flashlight-in-the-dark feel. Disabled on mobile.
Grain overlay: SVG fractal noise texture. Screen blend on dark, multiply on light. Adds analogue warmth to the digital surface.
Cursor: Crosshair on desktop. Reinforces the precision/builder identity.

09 — Usage rules

Do

Use the mark at provided proportions. Let it inherit currentColor in static contexts. Use accent teal for wings in digital contexts. Place on solid backgrounds only. Use the icon-only variant for small contexts (favicons, avatars, CLI output). Default to dark theme. Use Instrument Serif italic for editorial/prose copy alongside monospace UI text.

Don't

Don't rotate, stretch, or distort the mark. Don't place on busy or photographic backgrounds. Don't use colors other than monochrome or accent teal on the mark. Don't use the accent teal as a background fill or dominant color — it's for highlights only. Don't add drop shadows, outlines, or 3D effects. Don't add a tagline. Don't use Instrument Serif for UI elements, labels, or navigation — it's for prose only.

11 — Voice

dragonfly doesn't speak much. When it does:

Tone: Quiet, direct, technical. No exclamation marks.
Casing: All lowercase. "dragonfly" never capitalised.
Domain as name: When the full name is needed, use "dragonfly.haus" — it's the name and the address.
No tagline. No mission statement. No "about" page. The work speaks.
